## Охватываемая функциональность

### Bitmap операции
- Установка/проверка битов
- Граничные случаи (индексы 0, BitsPerPage-1)
- Сериализация/десериализация (ToBytes/FromBytes)
- Валидация входных данных

### Структура страниц (Page)
- Создание и инициализация
- Управление флагами (Dirty, WriteProtected)
- Отслеживание доступа (AccessTime, AccessCounter)
- Работа с bitmap'ом в контексте страницы
- Функциональность SetBit и IsBitSet

### LRU Кэш
- Базовые операции Put/Get
- Вытеснение при переполнении
- Обновление позиции при доступе
- Граничные случаи (capacity=1, множественные вытеснения)
- Проверка порядка вытеснения (Least Recently Used)

### Бинарный I/O
- Чтение/запись примитивных типов (int32, int64, byte)
- Работа с массивами байтов
- Множественные последовательные операции
- Граничные значения (MIN/MAX для типов)

### Обработка ошибок
- Коды ошибок (-1 до -8)
- Обёртывание ошибок (Unwrap)
- Сообщения об ошибках
- Правильность кодов для разных сценариев

### API операции
- Создание массивов разных типов (int, char, varchar)
- Операции Read/Write с валидацией индексов
- Управление множественными handle'ами
- Работа с кэшем (SetCacheSize, GetCacheSize)
- Граничные случаи (отрицательные индексы, вне диапазона)
- Помощь и статистика

## Использованные шаблоны и лучшие практики

1. **Table-driven tests** для проверки множественных случаев
2. **Cleanup гарантирование** через defer
3. **Граничные значения** - всегда проверяются экстремальные случаи
4. **Изоляция тестов** - каждый тест независим
5. **Понятные сообщения об ошибках** с ожидаемыми/полученными значениями
6. **Параллельное выполнение** через использование *testing.T

## Качество кода

- Нет жёстких зависимостей между тестами
- Использование временных файлов с автоматической очисткой
- Проверка граничных случаев и условий ошибки
- Правильная обработка ошибок (error checking)
- Соответствие стандартам Go testing
